# Security Policy

## Reporting a Vulnerability

Please do not report security vulnerabilities in public issues.

Use this repository's GitHub **Security** tab and choose **Report a vulnerability** to submit a private vulnerability report. This uses GitHub Private Vulnerability Reporting and lets maintainers coordinate a GitHub Security Advisory when appropriate.

If **Report a vulnerability** is unavailable, create a minimal public issue requesting a private contact channel. Do not include vulnerability details, reproduction steps, credentials, private keys, user data, or any other sensitive information in that issue.

For a private report, include a clear description, reproduction steps, affected version or commit, impact, and any proposed mitigation. Do not include credentials, private keys, or user data.

## Maintainer Note

After creating the public repository, enable GitHub Private Vulnerability Reporting in **Settings** > **Code security and analysis**.

## Scope

Reports are welcome for the repository's source code, build tooling, bundled plugins, and released artifacts when they exist. Tickpad is Alpha software; a report may be resolved by a fix, a documented limitation, or a decision not to support the affected behavior yet.
