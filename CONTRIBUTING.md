# Contributing

Tickpad is Alpha software. Keep contributions focused, small, and compatible with the existing plugin and Electron boundaries.

## Before Opening A Pull Request

1. Create an issue first for non-trivial changes so the direction can be discussed.
2. Keep unrelated formatting and refactors out of the pull request.
3. Add or update tests when behavior changes.
4. Run the required checks from the repository root:

```bash
pnpm test
pnpm typecheck
pnpm build
```

## Pull Requests

Describe the user-visible change, testing performed, and any remaining limitations. For UI changes, include screenshots when they clarify the result.

By contributing, you agree to follow the [Code of Conduct](CODE_OF_CONDUCT.md).
